Architecture

How MuseFlow Works

MuseFlow uses a Sequence-to-Sequence approach: it takes a melody as input and generates a full multi-instrument band arrangement conditioned on a genre tag.

🎤

Input: Melody Prompt

A hummed recording or MIDI file is tokenized into a 128-token melody prompt using the REMI tokenizer. Short melodies are automatically looped to fill the full context window.

🎸

Output: Band Arrangement

The Transformer generates up to 384 tokens of multi-instrument arrangement, including program changes, velocities, chords, and tempos — all genre-conditioned.

🧠

Transformer Core

A 4-layer, 8-head causal Transformer encoder with 256-dim embeddings. Trained with AdamW, cosine LR schedule, and mixed-precision (FP16) for efficient GPU usage.

🎛️

Genre Conditioning

Three special genre tokens (10000, 10001, 10002) act as style switches, telling the model to generate Classical, Lofi, or Rock arrangements.

Pipeline

End-to-End Pipeline

From raw MIDI files to AI-generated band arrangements in five stages.

1
download_dataset.py

Download & Classify Dataset

Downloads the Lakh MIDI Clean dataset (~223 MB, ~6,000+ MIDI files), then filters and classifies them into three genres.

  • Multi-instrument filter: ≥2 tracks, ≥2 distinct programs, 30–15,000 notes
  • Genre classification via scoring heuristic on MIDI program numbers
  • Targets: 🎻 Classical 🎹 Lofi 🎸 Rock — 500 files each
2
rescue.py

Rescue Single-Track Lofi Files

Some Lofi MIDI files are squished into a single track. The rescuer splits them into multi-instrument arrangements by pitch register:

  • ≥72 — Melody → Electric Piano (Prog 4)
  • 48–71 — Pad/Chords → Warm Pad (Prog 89)
  • <48 — Bass → Fingered E.Bass (Prog 34)
3
preprocess.py

Tokenize into Seq2Seq Pairs

Each MIDI file is split into a Melody (highest-pitched track) and a Band (all remaining tracks). Both are converted to REMI tokens, with a genre tag injected at the start of the band sequence.

  • Melody = prompt (128 tokens), Band = target (384 tokens)
  • Genre IDs: 10000 (Classical), 10001 (Lofi), 10002 (Rock)
  • Output: JSON files in tokens_seq2seq/
4
train.py

Train the Transformer

The Seq2Seq pairs are fed into the Transformer model. The model learns to predict each next token in the combined [Melody | GenreTag | BandArrangement] sequence.

  • 25 epochs with cosine annealing LR schedule
  • Mixed-precision (FP16) for ~50% VRAM savings
  • Gradient clipping at max_norm=1.0
  • Output: museflow_translator.pth (~40 MB)
5
generate.py

Generate Band Arrangements

Feed a melody + genre choice into the trained model. It auto-regressively generates a full multi-instrument arrangement with creative sampling and humanized durations.

  • Melody looping — short inputs are tiled to fill the 128-token window
  • Top-k (50) + nucleus (p=0.95) sampling at temperature 0.9
  • Duration humanizer adds ±15% jitter for a natural feel
  • Output: AI_Band_{GENRE}_Output.mid

Model Details

The Transformer Engine

Understanding the training data format, model internals, and generation strategy.

📊 Training Data Format

Each training sample is a 512-token sequence formed by concatenating:

SEQUENCE FORMAT // Total length: 128 + 384 = 512 tokens [ Melody Tokens (128) | Genre Tag (1) | Band Tokens (383) ] // Example genre tags: 10000 → Classical | 10001 → Lofi | 10002 → Rock

The model learns via next-token prediction with cross-entropy loss (padding index 0 is ignored). This means it learns to predict the band arrangement given the melody context.

🎲 Sampling Strategy

During inference, greedy decoding is replaced with a sophisticated sampling pipeline:

  • Temperature = 0.9 — Slightly sharpens the probability distribution
  • Top-k = 50 — Limits candidates to the 50 most likely tokens
  • Nucleus (top-p) = 0.95 — Further trims to the 95% cumulative probability mass

This allows rare but musically important tokens (like program changes for new instruments) to appear naturally.

🥁 Duration Humanizer

After decoding tokens back into MIDI, a humanization pass adds natural feel:

  • ±15% duration jitter — Each note's length varies slightly
  • ±5% onset drift — Notes start slightly early or late
  • Prevents the band from sounding rigidly quantized
  • Creates a natural "human performance" feel
Getting Started

Setup & Installation

Everything you need to get MuseFlow running on your machine.

💡
Requirements: Python 3.9+, CUDA-compatible GPU recommended (tested on RTX 3050 with 6 GB VRAM). CPU training works but is significantly slower.

1. Create a Virtual Environment

Terminal $ python -m venv env $ env\Scripts\activate # Windows $ source env/bin/activate # macOS / Linux

2. Install Dependencies

Terminal $ pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121 $ pip install symusic miditok basic-pitch music21
✅
Verify GPU: Run python test_gpu.py to check that CUDA is detected. You should see torch.cuda.is_available() = True.

3. Run the Full Pipeline

Terminal # Step 1: Download & classify MIDI dataset $ python download_dataset.py # Step 2: (Optional) Rescue single-track Lofi files $ python rescue.py # Step 3: Tokenize into Seq2Seq pairs $ python preprocess.py # Step 4: Train the Transformer (~25 epochs) $ python train.py # Step 5: Generate music! $ python generate.py # Or with a custom melody file: $ python generate.py hum_basic_pitch.mid
⚠️
First run: The dataset download is ~223 MB and extraction + classification can take 10–20 minutes. Training takes roughly 15–30 min on an RTX 3050.
Usage

Generating Music

How to use MuseFlow to turn your melodies into full band arrangements.

🎤 Option A: From a Hum / Audio

Record yourself humming into hum.wav, then transcribe it to MIDI and generate:

$ python transcribe.py # Creates hum_basic_pitch.mid $ python generate.py hum_basic_pitch.mid # Choose genre → outputs AI_Band_GENRE_Output.mid

🎹 Option B: From an Existing MIDI

Pass any MIDI file directly as the melody input:

$ python generate.py my_melody.mid # Interactive prompts ask for genre (classical/lofi/rock) # and loop count (1-8 bars)

🎛️ Generation Parameters

During generation, you'll be asked two questions interactively:

Prompt Options Effect
Genre classical / lofi / rock Selects the style of the band arrangement
Loop Count 1 – 8 Controls song length (~150 tokens per loop)

🎯 Genre Classification Heuristic

The classifier scores each MIDI file across three genre dimensions based on its General MIDI program numbers:

🎻 Classical

Strings (Prog 40–51) +4
Woodwinds (Prog 64–79) +4
Brass (Prog 56–63) +3
Strings + Woodwinds combo +3
Strings without drums/guitar +3

🎹 Lofi

Electric Piano (Prog 4–5) +5
Pads (Prog 88–95) +4
Vibraphone (Prog 11–14) +3
E.Piano + Bass combo +3
Synth Lead / Pads +2

🎸 Rock

Electric Guitar (Prog 25–31) +6
Distortion (Prog 29–30) +5
Drums + E.Guitar combo +3
Bass + E.Guitar combo +2
Organ + E.Guitar combo +2
